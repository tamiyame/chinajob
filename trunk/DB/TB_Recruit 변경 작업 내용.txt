TB_Recruit 테이블 변경 관련 작업

	InputLanguageType 네이밍변경 ---> CountryNo 으로 변경
	BenefitsSeverancePay 문자로 받을수 있게 변경 ---> nvarchar(20) 으로 변경
	BenefitsSeverancePayEtcValue 필요없는 컬럼 ---> 컬럼 삭제
	RecruitNo 이거 자동 증가 빼기 ---> IDENTIRY 제거

	primary key
		CountryNo, RecruitNo


	### 테이블 삭제 하고 새구조로 다시 만들어서 이전 데이터 넣었음...


1. CountryNo, RecruitNo 가 기본 조건 임! 모든 sp 입력 파라미터 순서가 @CountryNo, @RecruitNo 순서로 변경 되었음

2. [TSP_RecruitCreate] sp --- RecruitNo 컬럼 자동증가 제거로 Insert Query 변경되었음
	입력 파라미터는 모두 컬럼변경 따라 변경

	INSERT	TB_Recruit
	(
		CountryNo
	,	RecruitNo
	순으로...

3. [TSP_RecruitGetInfo] sp
	입력 파라미터 컬럼변경 따라 함께 변경

	WHERE절도 변경
	WHERE
		@CountryNo = CountryNo
	AND	@RecruitNo = RecruitNo


4. [TSP_RecruitGetList] sp
	내부 SELECT 컬럼 변경

5. [TSP_RecruitModify] sp
	입력 파라미터 컬럼변경 따라 함께 변경
	Update Query의 WHERE절 변경...CountryNo 추가
